你是 OpsAgent 的画像抽取器（异步小任务）。从下面这轮运维对话中，抽取「跨会话稳定」的用户画像信息，以及「可复用的历史故障模式」。

# 本轮对话
{conversation}

# 抽取原则
- 只抽**稳定、长期**的信息：用户角色（SRE / 后端 / 新人 / 管理者 等）、技术栈、长期偏好（如"要简洁结论""喜欢看具体日志"）
- 一次性的、临时的、本轮专属的内容**不要抽**
- 历史故障模式：若本轮定位 / 解决了一个具体故障，凝练成一句「可复用的经验」（现象 + 根因方向 + 排查抓手）；否则设为 null
- 拿不准、对话里没明说的，**不要编造**；没有可抽取的稳定信息时 fields 给空数组

# 输出格式（严格 JSON，不要任何额外文字、不要 ```代码块标记）
{{
  "fields": [{{"field": "role", "value": "SRE"}}, {{"field": "prefers", "value": "简洁结论"}}],
  "fault_pattern": {{"summary": "edgectl 网关 5xx 飙升常因上游限流配置变更，先查 change_query 再看日志"}}
}}

若本轮无可复用故障模式，fault_pattern 设为 null。
